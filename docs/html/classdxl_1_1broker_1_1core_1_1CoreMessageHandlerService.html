Detailed Description

Service for registering message handler instances.

These handlers will be called prior to a message being published and/or stored for sending to a particular recipient on a specified message topic.

The flow of a message is as follows:

onPublish - The message is about to be published. At this point, the publishing can be vetoed.

onStore - The message has been stored in the core database for sending to recipients. At this point a CoreMessageContext object will be created. This message context object will exist until after the finalize method has been invoked.

onInsert - This is called for each of the possible recipients of the message (this queues for delivery to the recipient). The insertion can be vetoed.

onFinalize - The message sending process has completed. After this message is invoked, the message context object will be destroyed.
